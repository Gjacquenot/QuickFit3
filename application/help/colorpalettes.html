The plot widgets in QuickFit use a common set of color palettes that may be used to draw false-color images. A set of approx. 50 palettes is hardcoded in QuickFit. This set can be extended with color palette files in the directory $$assetsdir$$/colorpalettes/. Every palette is made up of a list of colors (as red, green and blue combonents r,g,b and an according positions x in the palette. The position has to be in increasing order. The values in between the given colors are linearly interpolated. E.g. the following palette would create a color scale starting at blue, changing to green and ending at red: 

  # x, r, g, b

  0.0, 0, 0, 255

  0.5, 0, 255, 0

  1.0, 255, 0, 0

If the x-column is changed, also the position of the defined colors changes: 
  # x, r, g, b

  0.0, 0, 0, 255

  0.25, 0, 255, 0

  1.0, 255, 0, 0

Several fileformats are allowed:

  • ParaView XML Format (*.xml): Palettes in this format are stored as XML file of the form: 
      <doc>
    
  <ColorMap name="example BGR" space="RGB">
    
    <Point x="0" r="0" g="0" b="1.0"/>
    
    <Point x="0.5" r="0" g="1.00" b="0"/>
    
    <Point x="1" r="1" g="0" b="0"/>
    
  </ColorMap>
    
  </doc>
    Several ColorMap tags are allowed per file, each defining a color map with the given name. Currently only the color space RGB is supported by QuickFit. All color values (r,g,b) have to be in the range 0..1.
  • Text palette format (*.csv *.pal *.rgb): This format is a simple comma/tab/space separated file format. The files may contain 3 (r,g,b) or 4 columns (x,r,g,b) and the RGB-values have to scale between 0..1 or 0..255 (the same on a per-file basis). If no x-column is given, x is linearly spaced. Here are two examples, which generate the two colorbars above: 
      # r g b
    
  0 0 1
    
  0 1 0
    
  1 0 0
    Or with XRGB: 
      # x, r, g, b
    
  0.0, 0, 0, 255
    
  0.25, 0, 255, 0
    
  1.0, 255, 0, 0
    The name of the palette will be the same as the filename.
