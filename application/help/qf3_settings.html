This dialog allows to set several options that change how QuickFit looks and operates.

  1. Network/Update Check
    • Network Proxy: Set the proxy settings that QuickFit will use, when connecting to the internet (mostly to check for updates, see below, see also the menu entry Help | Check for Updates ...).
    • check for updates on startup: If activated, QuickFit will automatically check for updates by connecting to the QuickFit homepage. If this is deactivated, you can still check manually with the menu entry Help | Check for Updates ...
  2. View, Language, General Settings
    • Language: If translations of QuickFit are installed you can select one here. Note that you will have to restart QuickFit before the translation takes full effect.
    • Style/Stylesheet: Here you can select different styling options for QuickFit
    • font for online-help: Here you can set the basic font and font-size used to display online-help pages.
    • font for math rendering: Here you can set the basic font-size used when rendering equations for the online-help.
    • RDR/evaluation editor window title: choose the text/information in the header of an evaluation/RDR dialog.
    • file dialogs: Choose which file-select dialogs to use. In some cases you might want to use the standardized internal dialogs, instead of the standard system dialogs. Note however, that these internal dialogs may not offer the full functionality you are used to by the system dialogs.
    • welcome screen: Here you can select whether to show the welcome screen on startup of QuickFit.
    • VAR-COV-matrices: Some plugins display the Var-COV-matrix from a fit and color-code its entries. This option allows to choose a non-linear color-coding. The color-coding uses a gamma(γ)-transfrom for the matrix entries, which have been normalized to $(v_{\pm1}\in[-1,1])$: $[v_{\pm1}'=\left|v_{\pm1}\right|^\gamma]$
  3. Parallelization
    • Maximum Number of Threads: Specifies how many threads may be started by QuickFit for parallel processing (optional). You should set this to a number equal or slightly smaller to the number of processor cores, your computer has.
  4. Autosave
    • autosave project every x minutes: specifies the autosave interval for the project
    • ask user to save project after first edit: If this option is activated, QuickFit will ask the user to save the crrent project, right after the first record/evaluation has been added.
  5. Debugging
    • show additional debug messages in main window: If this is activated, additional debug output (e.g. from the Qt library) is shown on the main window in an additional "Debug Log" tab. This debug outout is written to quickfit3_qdebug.log in the QuickFit user settings directory. If this function is deactivated these outputs are redirected to the standard console output. If you have stability problems with Quickfit, it might help to deactivate this function (especially on Linux).
  6. User Content/Directories
    • user fitfunctions folder\n(for parsed and library functions): Give a folder in which QuickFit will search for user-defined fit functions (either as a dynamically linked library (.dll / .so / .dylib) or as a parsed user fit function.
    • user settings folder: Choose the folder, where QuickFit stores the user-settings. usually this is .quickfit3 (hidden on some operating systems!) within the user's home directory.
    • global settings folder: Choose the folder, where QuickFit searches for global settings, such as for hardware-control plugins. This folder can also be read-only, so you can configure some plugins in QuickFit globally for all users of a computer.
    • temporary files folder: Here you can choose to overwrite the system's default temporary files folder. SOmetimes there are problems in gaining write-access to this folder. In these cases, you can choose another folder. $$warning:If there are no problems, it is NOT ADVISED to overwrite the system temporary folder!$$
    $$note:You might have to restart QuickFit, so a changed folder takes full effect (especially the configuration fodlers!).$$
  7. further configuration possibilities are provided by plugins and are documented with them: $$settings_contents$$
