The Help System

The Online-Help system of QuickFit is based on HTML. A subset of the full HTML spectrum of tags is supported. Some special "tags" are replaced before the page is displayed and some formatting and a table of contents is created automatically. The replaced "tags" are always enclosed in double-dollars, e.g. $$references$$ will be replaced by a reference list. After all replacements have been performed, the remaining $$TAGS$$ are removed. You may use the HTML-entity sequence &#36;&#36; to escape $$. This help page contains a list of all available replace tags.

Tag Reference

useful information:

  • $$mainhelpdir$$ the path (including a trainiling slash) to the main help directory

version information:

  • $$version.svnrevision$$ will be replaced by the SVN revision of the QuickFit
  • $$version.status$$ will be replaced by the application status (beta, alpha, final, ...)
  • $$version.date$$ will be replaced by the compile date
  • $$version$$ will be replaced by the version number (e.g. 3.0)
  • $$version_full$$ will be replaced by the full version string

copyright information:

  • $$thanksto$$ will be replaced by a list of people that our thanks go to for helping with QuickFit
  • $$copyright$$ will be replaced by a copyright note
  • $$author$$ will be replaced by QuickFit's author
  • $$weblink$$ will be replaced by the QuickFit webpage
  • $$weblink_source$$ will be replaced by the QuickFit source code repository

page footer:

  • $$qf_commondoc_footer.start$$ starts the page footer
  • $$qf_commondoc_footer.end$$ ends the page footer

page header:

  • $$qf_commondoc_header.start$$ starts the page header
  • $$qf_commondoc_header.end$$ ends the page header and adds a heading with the title-tag contents
  • $$qf_commondoc_header.end_notitle$$ ends the page header
  • $$qf_commondoc_header.rdr$$ link to basic info about raw data records
  • $$qf_commondoc_header.eval$$ link to basic info about evaluations
  • $$qf_commondoc_header.extension$$ link to basic info about extensions
  • $$qf_commondoc_header.fitfunc$$ link to basic info about fit functions
  • $$qf_commondoc_header.fitalg$$ link to basic info about fit algorithms
  • $$qf_commondoc_header.separator$$ separator between two entries in the header

plugin info: If in a plugin directory, these infos are available:

  • $$local_plugin_icon$$ displays plugin icon
  • $$local_plugin_iconfilename$$ displays plugin icon filename
  • $$local_plugin_name$$ displays plugin name
  • $$local_plugin_author$$ displays plugin author
  • $$local_plugin_copyright$$ displays plugin copyright message
  • $$local_plugin_weblink_url$$ displays plugin weblink url
  • $$local_plugin_weblink$$ displays plugin weblink as a link
  • $$local_plugin_id$$ displays plugin id
  • $$local_plugin_tutorial_file$$ displays plugin tutorial filename
  • $$local_plugin_tutorial_link$$ displays a link to the plugin tutorial
  • $$local_plugin_mainhelp_file$$ displays plugin main help page filename
  • $$local_plugin_mainhelp_link$$ displays a link to the plugin main help page
  • $$local_plugin_version$$ displays plugin version (major and minor)
  • $$local_plugin_typehelp_link$$ link to a helppage covering the basic plugin type
  • $$local_plugin_typehelp_file$$ file containing a helppage covering the basic plugin type

other plugins info:

  • $$plugin_info:help:PLUGINID$$ the path of the help file for plugin with ID PLUGINID
  • $$plugin_info:tutorial:PLUGINID$$ the path of the tutorial file for plugin with ID PLUGINID
  • $$plugin_info:faq:PLUGINID$$ the path of the frequently asked questions file for plugin with ID PLUGINID
  • $$plugin_info:helpdir:PLUGINID$$ the path of the helpdir file for plugin with ID PLUGINID
  • $$plugin_info:assetsdir:PLUGINID$$ the path of the assetsdir file for plugin with ID PLUGINID
  • $$plugin_info:configdir:PLUGINID$$ the path of the configdir file for plugin with ID PLUGINID
  • $$plugin_info:examplesdir:PLUGINID$$ the path of the examplesdir file for plugin with ID PLUGINID
  • $$fitfunction:help:PLUGINID$$ help file for the fit function ID PLUGINID
  • $$fitfunction:name:PLUGINID$$ name for the fit function ID PLUGINID
  • $$fitfunction:short_name:PLUGINID$$ short name for the fit function ID PLUGINID
  • $$fitalgorithm:help:PLUGINID$$ help file for the fit algorithm ID PLUGINID
  • $$fitalgorithm:name:PLUGINID$$ name for the fit algorithm ID PLUGINID
  • $$fitalgorithm:short_name:PLUGINID$$ short name for the fit algorithm ID PLUGINID
  • $$importer:help:PLUGINID$$ help file for the importer ID PLUGINID
  • $$importer:name:PLUGINID$$ format name for the importer ID PLUGINID
  • $$exporter:help:PLUGINID$$ help file for the importer ID PLUGINID
  • $$exporter:name:PLUGINID$$ format name for the importer ID PLUGINID

lists of available plugins

  • $$list:extension:INTERFACENAME$$ makes a list (unordered HTML list) of all available extensions of type INTERFACENAME (can also be empty, then the list contains all extensions)
  • $$list:fitfunc:STARTSWITH$$ makes a list (unordered HTML list) of all fit functions that start with STARTSWITH (can also be empty, then the list contains all fit functions)
  • $$list:fitalg:$$ (note the second colon!!!) makes a list (unorder HTML list) of all fit algorithms
  • $$tutorials_contents$$ will be replaced by a list of plugin tutorials (complete with links as unordered list!)
  • $$help_contents$$ will be replaced by a list of plugin main help files (complete with links as unordered list!)

insert file's contents

  • $$insert:FILENAME$$ insert the contents (complete !!!) of the given file FILENAME (path relative to current directory)
  • $$insertglobal:FILENAME$$ insert the contents (complete !!!) of the given file FILENAME in the global help directory

insert LaTeX formulas

  • $$math:LATEX$$ renders the given LaTeX fragment using a LaTeX parser and displays the result. Note that the rendering quality of this is not nearly as good as the quality output by LaTeX. So if you want high-quality output, or don't like a formula rendered with $$math:...$$ or need a LaTeX feature not supported by this, please include a direct LaTeX rendering result as PNG.
  • $$bmath:LATEX$$ displays an indented block of rendered LaTeX.

go back to top bar:

  • $$qf_commondoc_backtop$$ displays a blue bar with an upward arrow that allows to jump back to the top of the page

literature reference tags:

  • $$ref:ID:REFERENCE TEXT$$ will be relaced by a bracketed reference number [2] that links to the references list, introduced by $$references$$ into the document. Both parameters are strictly optional: If ID is missing the reference is single, if REFERENCE TEXT is missing, but an already defined ID is present, the ref will link to the specified (and predefined) reference.
  • $$invisibleref:ID:REFERENCE TEXT$$ works like above, but no bracketed number is introduced. The referenced literature will still be introduced into the referneces list.
  • $$references$$ will introduce a list of references into the document.

Additional Commands:

  • $$see:text$$ will add a yellow box with the test "See" and the given text to the help document
  • $$note:text$$ will add a blue box with the test "Note" and the given text to the help document
  • $$info:text$$ will add a blue box with the test "Information" and the given text to the help document
  • $$warning:text$$ will add a orange/red box with the test "Warning" and the given text to the help document
  • $$example:text$$ will add a blue/gray box with the text formatted as example
  • $$cexample:text$$ / $$codeexample:text$$ will add a blue/gray box with the text formatted as a code example (typewrite/Courier font)
  • $$tt:text$$ / $$code:text$$ will format the text in the typewrite/Courier font
  • $$fig:file:caption$$ / $$figure:file:caption$$ will add the given image file as a figure with the given caption

Special files in online-help

localreplaces.ini

You can specify your own $$...$$ commands for a whole directory, by creating an ini-file named localreplaces.ini, which contains lines like COMMAND_NAME="replacedtext". Then QuickFit will replace all occurences of $$COMMAND_NAME$$ by the given text.

tooltips.ini

You can specify tooltips, which will be used globall by the QuickFit online-help system, by adding a file tooltips.ini to your plugin online-help directory. This file is organized as an INI-file, as follows: 

RDR="<b>raw data record</b>, an intem in the QuickFit project tree, representing some raw data (e.g. correlation curves, an image stack, ...)"
raw data record=%RDR
rdr group="Raw data records can be grouped in oder to tell QuickFit that several RDRs belong together. e.g. for an FCCS measurement the two autocorrelations and the crosscorrelation data will be grouped."
TIFF="<b>tagged image file format (TIFF)</b>: a fileformat to store images or series of images"
The lhs of each line (before the =) contains the text, that should be clickable afterwards. The rhs (after the =), the tooltip is written in quotation marks. If you need to use quotation marks within you tooltip, you need to escape them as \". QuickFit will transfrom the first match of the lhs in each help file into a tooltip-link. SO if the user clicks on that text, a sub-window with your tooltip will be displayed. Since often several different writing should all lead to the same tooltip, there is a short cut: You can define the tooltip with a lhs (e.g. RDR in the example above) without whitespaces and special characters. Then you can reuse that tooltip, by writing %RDR on the rhs.

autolinks.ini

Autolinks work a bit like tooltips, but they do not show up as a hovering sub-window, but as a small image () in the text, which links to a special help page for the topic. You can add new autolinks, by adding a file autolinks.ini to you project, which has the following format: 

expression parser="mathparser.html"
plotter widget="jkqtplotter.html"
regular expressions="$$mainhelpdir$$qf3_qtregexp.html"
All the same as for tooltips, the lhs defines the word(s) to match and the rhs the file to link to. You can use the $$mainhelpdir$$-macro to specify files in the main help directory. If the path does NOT start with a $$, it is interpreted as a relative path within the help directory of your plugin. The %... shortcut also works for auto-links.

tutorials.ini

If you plugin comes with only one tutorial, you can simply generate a file tutorial.html, which contains the tutorial. If there are several tutorials, you might want them to be listed in QuickFi's main tutorial page. Then you will have to put each tutorial in it's separate file, have tutorial.html link to each of these files and generate an additional file tutirals.ini, which summarizes all your tutorials in the following format: 

[tutorial1]
name=Tutorial for Sub 1
file=tutorial_sub1.html

[tutorial2]
name=Tutorial for Sub 2
file=tutorial_sub2.html

Expression parser reference files

If your plugin implements functions for QuickFit's expression parser, you should also add documentation for these functions. This documentation has to be stored in a special format, which allows QuickFit to parse it. You need to do several things:

  1. generate a sub-directory parserreference, which will contain several HTML help files, which describe the fit fucntions (each file may also describe a group of functions).
  2. Directly in the help directory, generate a file parserref.ini with the following format: 
       
  <li><b><a href="$$plugin_info:helpdir:PLUGINID$$/parserreference/FILENAME1.html"></a></b></li>
  <li><b><a href="$$plugin_info:helpdir:PLUGINID$$/parserreference/FILENAME2.html"></a></b></li>
    which lists all HTML help pages, that you generated in parserreference (here FILENAME1.html, FILENAME2.html). Also make sure to replace PLUGINID by the ID of your plugin! With this file present, the help files will be listed in the main expression parser help.
  3. In the main help of your plugin, you can add a line like <ul>$$insert:parserref.inc$$</ul> to generate a list of your help files.
  4. Now generate the sub-help files in parserreference, i.e. FILENAME1.html and FILENAME2.htmlin the example above. Each file should list the description of one or more of the functions and constants, that you added to the expression parser. The default form is: 
    $$funcref_start$$<a name="FUNC_NAME"/><!-- func:FUNC_NAME --><b><tt><!-- template -->FUNC_NAME(x)<!-- /template --></tt> - <i> compute BLA of parameter <tt>x</tt> </i>:</b>
$$funcref_description$$
    MORE ELABORATE DESCRIPTION.
        $$example:The call <tt>FUNC_NAME([1,2,3])</tt> will return <tt>BLA</tt>$$  
<!-- /func:FUNC_NAME -->
$$funcref_end$$

Frequently asked questions page

Each plugin may also provide a list of FAQs. This is done in a single help HTML-file faq.html. For each FAQ, it has to contain an entry like this: 

$$faq_start$$
  <a name="FAQ1"><b>How to ...?</b>
$$faq_answer$$
  Answer ...
$$faq_end$$
All entries of this format will be recognized by QuickFit and the FAQs will be listed in QuickFit's main FAQ page.

Metadata
