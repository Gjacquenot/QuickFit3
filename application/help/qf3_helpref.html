The Help System

The Online-Help system of QuickFit is based on HTML. A subset of the full HTML spectrum of tags is supported. Some special "tags" are replaced before the page is displayed and some formatting and a table of contents is created automatically. The replaced "tags" are always enclosed in double-dollars, e.g. $$references$$ will be replaced by a reference list. After all replacements have been performed, the remaining $$TAGS$$ are removed. You may use the HTML-entity sequence &#36;&#36; to escape $$. This help page contains a list of all available replace tags.

Tag Reference

useful information:

  • $$mainhelpdir$$ the path (including a trainiling slash) to the main help directory

version information:

  • $$version.svnrevision$$ will be replaced by the SVN revision of the QuickFit
  • $$version.status$$ will be replaced by the application status (beta, alpha, final, ...)
  • $$version.date$$ will be replaced by the compile date
  • $$version$$ will be replaced by the version number (e.g. 3.0)
  • $$version_full$$ will be replaced by the full version string

copyright information:

  • $$thanksto$$ will be replaced by a list of people that our thanks go to for helping with QuickFit
  • $$copyright$$ will be replaced by a copyright note
  • $$author$$ will be replaced by QuickFit's author
  • $$weblink$$ will be replaced by the QuickFit webpage

page footer:

  • $$qf_commondoc_footer.start$$ starts the page footer
  • $$qf_commondoc_footer.end$$ ends the page footer

page header:

  • $$qf_commondoc_header.start$$ starts the page header
  • $$qf_commondoc_header.end$$ ends the page header and adds a heading with the title-tag contents
  • $$qf_commondoc_header.end_notitle$$ ends the page header
  • $$qf_commondoc_header.rdr$$ link to basic info about raw data records
  • $$qf_commondoc_header.eval$$ link to basic info about evaluations
  • $$qf_commondoc_header.extension$$ link to basic info about extensions
  • $$qf_commondoc_header.fitfunc$$ link to basic info about fit functions
  • $$qf_commondoc_header.fitalg$$ link to basic info about fit algorithms
  • $$qf_commondoc_header.separator$$ separator between two entries in the header

plugin info: If in a plugin directory, these infos are available:

  • $$local_plugin_icon$$ displays plugin icon
  • $$local_plugin_iconfilename$$ displays plugin icon filename
  • $$local_plugin_name$$ displays plugin name
  • $$local_plugin_author$$ displays plugin author
  • $$local_plugin_copyright$$ displays plugin copyright message
  • $$local_plugin_weblink_url$$ displays plugin weblink url
  • $$local_plugin_weblink$$ displays plugin weblink as a link
  • $$local_plugin_id$$ displays plugin id
  • $$local_plugin_tutorial_file$$ displays plugin tutorial filename
  • $$local_plugin_tutorial_link$$ displays a link to the plugin tutorial
  • $$local_plugin_mainhelp_file$$ displays plugin main help page filename
  • $$local_plugin_mainhelp_link$$ displays a link to the plugin main help page
  • $$local_plugin_version$$ displays plugin version (major and minor)
  • $$local_plugin_typehelp_link$$ link to a helppage covering the basic plugin type
  • $$local_plugin_typehelp_file$$ file containing a helppage covering the basic plugin type

other plugins info:

  • $$plugin_info:help:PLUGINID$$ the path of the help file for plugin with ID PLUGINID
  • $$plugin_info:tutorial:PLUGINID$$ the path of the tutorial file for plugin with ID PLUGINID

lists of available plugins

  • $$list:extension:INTERFACENAME$$ makes a list (unordered HTML list) of all available extensions of type INTERFACENAME (can also be empty, then the list contains all extensions)
  • $$list:fitfunc:STARTSWITH$$ makes a list (unordered HTML list) of all fit functions that start with STARTSWITH (can also be empty, then the list contains all fit functions)
  • $$list:fitalg:$$ (note the second colon!!!) makes a list (unorder HTML list) of all fit algorithms
  • $$tutorials_contents$$ will be replaced by a list of plugin tutorials (complete with links as unordered list!)
  • $$help_contents$$ will be replaced by a list of plugin main help files (complete with links as unordered list!)

insert file's contents

  • $$insert:FILENAME$$ insert the contents (complete !!!) of the given file FILENAME (path relative to current directory)
  • $$insertglobal:FILENAME$$ insert the contents (complete !!!) of the given file FILENAME in the global help directory

insert LaTeX formulas

  • $$math:LATEX$$ renders the given LaTeX fragment using a LaTeX parser and displays the result. Note that the rendering quality of this is not nearly as good as the quality output by LaTeX. So if you want high-quality output, or don't like a formula rendered with $$math:...$$ or need a LaTeX feature not supported by this, please include a direct LaTeX rendering result as PNG.
  • $$bmath:LATEX$$ displays an indented block of rendered LaTeX.

go back to top bar:

  • $$qf_commondoc_backtop$$ displays a blue bar with an upward arrow that allows to jump back to the top of the page

literature reference tags:

  • $$ref:ID:REFERENCE TEXT$$ will be relaced by a bracketed reference number [2] that links to the references list, introduced by $$references$$ into the document. Both parameters are strictly optional: If ID is missing the reference is single, if REFERENCE TEXT is missing, but an already defined ID is present, the ref will link to the specified (and predefined) reference.
  • $$invisibleref:ID:REFERENCE TEXT$$ works like above, but no bracketed number is introduced. The referenced literature will still be introduced into the referneces list.
  • $$references$$ will introduce a list of references into the document.

Metadata

In the default header some of the metadata from the HTML page are used to display buttons to go to the next, previous and contents page:

<title>SPIM Plugin: Preview Windows</title>
<link rel="prev" href="ext_spimb040_camconfig.html">
<link rel="next" href="ext_spimb040_stageconfig.html">
<link rel="contents" href="ext_spimb040.html">
