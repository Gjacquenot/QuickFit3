QuickFit then creates a new (or appends to an existing) table RDR in the current project, adds all data columns, that are necessary for the plot to that table and creates a plot in it, which resembles the plot you want to save. This way, the plot can be altered and formatted (e.g. for a publication). ALso the data can be procesed further (fits, statistics etc.).
To copy a plot in this way,
  1. right-click the plot
  2. choose copy plot data to table RDR
  3. set all properties in the following dialog as required (e.g. give a new name for the table-RDR, or select an existing one and give a new name for the plot in the table (or select an existing one).
  4. now you can edit the plot (and its data), by double-clicking the new table RDR in the project tree.

A detailed description can be found under $$qf_ui_jkqtplotter_helpfiletitle$$. $$faq_end$$ $$faq_start$$ How can I export the graph data used for a plot in QuickFit? $$faq_answer$$ Most plots in QuickFit allow to access the data, which was used to generate them. You have generally several possibilities, that are all accessible in the context-menu (pops open after a right-click) of the plot:
  1. choose Save Data, which saves the data for the plot as a table into a file (e.g. in the CSV, Excel, Matlab, ... format).
  2. choose Copy Data, which copies the data for the plot as a table into the clipboard.
  3. choose Copy Data Matlab, which copies the data for the plot as a Matlab script into the clipboard.
  4. choose Show Plot Data, which opens a new window, which contains the plot data as a table
  5. copy the plot data to a table RDR, which then contains the plot data as table.

See also: $$qf_ui_jkqtplotter_helpfiletitle$$. $$faq_end$$ $$faq_start$$ How can I export data to Matlab(r)? $$faq_answer$$ Generally you can use the context-menus to export the data of any plot (right-click the plot and choose Save Data or Show Plot Data and save from the table that opens up)
and of any data table (select the data to export, right-click and choose Save Selection | Save Selection, or any of the other Save Data options).
Then you can choose different file formats to save to.
Among these there are several options for Matlab:
  • Matlab MAT files (*.mat) stores the data as a (binary) MAT-file that can be loaded in Matlab or GNU Octave using the instruction
    load('filename', '-v6')
    The options "as Cell/Matrix" try to save the data as a double-matrix, but if this does not succeed (e.g. because the table also contains text), data is stored as a cell array. This retains all data in the table. The options "as Matrix" convert data to a number-matrix and remove any text or other data that cannot be exported. Such cells will be saved as NAN (not a number).
  • Matlab script (*.m) save a Matlab script that contains the single definition of a matrix/cell array with the data in plain text format. This is a simplified option, also used when "copy data to Matlab" is choosen from the table context menu.
