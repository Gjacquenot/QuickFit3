At many places inside QuickFit plots are used. Many of these plot displays use the same source code in order to draw the plots. They also feature a basic set of possibilities to interact with the user. Note however, that some of these possibilities may be deachtivated at some places, or you find them somewhere else. In those cases the online-help will explain what is not usable. Commonly all plots support these features:

Toolbar

If you move the mouse to the top left corner of the plot widget, a toolbar will appear (and disappear, if the mouse moves away from it). It provides several possibilities:

  • Save Plot: saves the plot as displayed on the screen as image (PNG,BMP, JPEG, ...), PostScript or PDF file
  • Print Plot: print the plot
  • Save Data: saves the data used to draw the plot as a CSV, SYLK or DIF files which may be read by other programs (like scientific plot programs, spreadsheet applications, ...)
  • Copy Data to Clipboard: copies the data used for the plot(s) to the clipboard, so it can be pasted into spreadsheet programs like excel as a table
  • Copy Data to Clipboard for Matlab: copies the data used for the plot(s) as a Matlab script to the clipboard. Each column is output as a separate vector in the script with a unique name
  • Show Plot Data: opens a new window, which contains the plot data as a table
  • Zoom All: reset the zoom to a state where all elements of the plot are visible
  • Zoom In: zoom into the center of the plot
  • Zoom out: Zoom out of the plot
  • copy plot data to table RDR: This function can be used to create a table raw data record from the current plot. The record will contain all data necessary to create the plot and the plot itself. This alows to easily save a certain plot inside the project.
    After activating this function, a dialog will open that allows to select what data is copied and where it is copied:
    First you will have to choose, whether to add the data (and the plot) to an existing table or whether to create a new one.
    Then you can choose a prefix, which is added to every column inserted into the table. This is especially useful, if several plots should be copied into one table, to later distinguish the different data columns.
    Finally you can choose whether only the data should be added, or whether also the plot should be copied to the table. In the first case, the plot data will only be added to the table, in the second case, a new plot will be created, or an existing plot will be extended. $$note:The data will be copied and is NOT linked to the plot, so if the plot changes afterwards, the table RDR data stays the same!$$
  • copy graph(s) to curve RDR: This function can be used to create a curves raw data record from the current plot. This is not possible for all graphs in the plot, only for those graphs that are based on XY-data (with errors). Especially it is not possible for image plots!
    After clicking on this menu entry, you will be asked to select the curves to export (if there are more than one) and then a new curve RDR will be added to the project. $$note:The data will be copied and is NOT linked to the plot, so if the plot changes afterwards, the curve RDR data stays the same!$$

Zooming

You can zoom into the graphs on several ways:

  • You can draw a zoom rectangle onto the plot, by pressing the left mouse button on the first corner and then pull the rectangle to the desired size. Then you have to release the left mouse button and the plot range will change accoring to your selection
  • You can use the mousewheel to zoom into the plot. The plot range will be expanded/reduced around the current mouse cursor position as the new plot center.
