Basic User Interface

Window

This page describes the basic features any raw data record screen implements.

The screen shows at least three tabs at the top. The button Help may be used to display this online help message.

In the head-line of this dialog you will find two buttons and that allow to move to the next and previous record of the same type. The button allows to delete the current record from the project. Between the buttons QuickFit displays an icon representing the type of the current record and also its name.

Properties

The first tab Properties (see image above) may be used to enter these:

  • the internal unique ID (an integer number) of the raw data record
  • a desciption of the evaluation type
  • a name for the raw data record (which will also appear in the project tree)
  • a group to which the RDR belongs, Select "--- new group ---" to create a new group. A dialog will pop up into which the name for the new group is entered.
  • a role: Each RDR may have a certain role within the project. this can e.g. be used to specify whether an FCS correlation record represents an ACF or a CCF
  • a description text. In the description you may e.g. note how the data was taken, which experiment you did, which instruments you used, ...
  • a path (with slash '/' as separator), which is used to group the records in the project tree. Simply enter a path like "mypath/subfolder" and the record will be inside the folders mypath and the subfolder below the raw data record nodes.
  • a list of all files associated with the record. Although these files are shown with their absolute path, only relative pathes are saved in the project where possible (e.g. when the files are on the same drive as the project file on a windows system).
  • a list of properties. A property is key/name (string) together with an arbitrary value (string, integer,floating-point number, boolean) that may e.g. be used to describe some additional information about the record, or the ambient conditions of the experiment, or any metadata, you like. These values are readable by evaluations. You may add a new property using the button to add a new property and to remove the selected proeprty. You may edit the property value directly in the table, but note that some properties may be system-properties and are not editable. Any property you added by hand is editable in any case. To start editing of the parameter name, type or value, double-click into a table cell.
    It is also possible to make a non-editable property editable, using the properties context menu. But note that this changed property may be overwritten when the project is loaded again, as read-only properties are often used to display data read from input files!

Evaluation Results

The second last tab Evaluation Results may be used to view the results of all evaluation objects that were used together with the current raw data record. These results (but only for one evaluation item) are also visible on the evaluation screen.

  • The results are shown as a table, where every row represents the set of results for ONE raw data record. Every row is designated by the evaluation result id, which consists of the raw data record name (e.g. 20nM_488_Day2_04s.asc - CH0), the evaluation type name together with the internal ID of the evaluation object (e.g. fcs_fit_1 where 1 is the ID) and an evaluation-dependend suffix (e.g. fcd_diff1_run0) . Every column shows the single results (if they are numbers or strings).
  • average and standard deviation over the currently selected cells will be sidplayed as a second table below the results table.
  • If a cell contains not a single value or value/error pair it might not be possible to display it in a concise form. Usually vectors/matrices are only displayed directly, if they contain less than 10 elements. If there are more items only the data type name is displayed to save space. If the mouse hovers of a vector/matrix field, the average value and standard deviation of the numbers in the list (if applicable) is shown as a tooltip.
  • reloads the results. This may be necessary, when new fit results were produced, but the table was not yet updated with them.
  • deletes the selected results. You can also click on the top-left corner of the table, which will select all results and then click on . This will delete all results from the current evaluation in the current project.
  • Many functions are also accessible over the windows main menu and the tables context menu.

Filtering results/Change displayed rows&columns

The results table can be filtered and modified interactively:

  • The evaluation filter allows to interactively filter the columns in the table: If this is non-empty, only rows that match (match means "contains", wildcards * for many characters and ? for a single character are possible) the filter string are displayed. If "RegExp" is activated you may use full regular expressions instead of the simple wildcards. The and not contains field allows to define a filter, which should NOT be contained in the column name.
  • The result filter allows to filter the rows, the same way as the evaluation filter filters the columns. If "RegExp" is activated you may use full regular expressions instead of the simple wildcards.
  • Results may also be in vector or matrix form. For these two display modes are available, which may be changed with the checkbox "show avg+/-SD for vector/matrix results": If it is activated, the average and standard deviation over the full vector/matrix is displayed (in blue and italic). Also the datatype is named in the second line in each cell. If this option is deactivated, only the type name is displayed if more than 10 entries are present. Otherwise the entries are displayed as a list.
    $$example:If you only want to display the fit statistics and not the results, you can e.g. enter fit statistics as a result filter. If you're not interested in the fix-columns, which indicate, whether the fit parameter was fixed, you can enter fix in the and not contains field.$$

Export/Print/Copy data

Most of these functions are found in the context menu of the table. Some are also accessible with buttons above the table. These functions always apply to all cells:

  • Print Table prints the current table
  • context-menu: Copy table as image copies the table as a pixel image/bitmap into the clipboard. You can then paste the table into any image or text processing software.
  • save all results as file stores the complete table as a data file (e.g. CSV, Excel, Matlab, ...). If any result is a vector or matrix (displayed as a single cell in the table), it is expanded in the file.
  • Save all results to file, averaged vector/matrix results stores the complete table as a data file (e.g. CSV, Excel, Matlab, ...). If any result is a vector or matrix (displayed as a single cell in the table), only the average and standard dveiation are stored in the file.
The following functions apply only to the currently selected cells in the table. If you want to save/copy all cells, you have to select them by clicking on the top-left corner of the table. They are also mostly found in the context menu:
  • Copy Selection as value+error pairs copies the selected cells as a table into the clipboard. If a result contains an error, two columns are copied and if it is a vector/matrix, the average and standard deciation are calculated.
  • Copy Selection as value+error pairs, w/o header copies the selected cells as a table into the clipboard. If a result contains an error, two columns are copied and if it is a vector/matrix, the average and standard deciation are calculated. This instruction omits the column/row headers, i.e. copies the data only.
  • Copy Selection as median+q25+q75 copies the selected cells as a table into the clipboard. If a result is a vector/matrix, the median, the 25% and the 75% quantile are calculated.
  • Copy Selection as median+q25+q75, w/o header copies the selected cells as a table into the clipboard. If a result is a vector/matrix, the median, the 25% and the 75% quantile are calculated. This instruction omits the column/row headers, i.e. copies the data only.
  • copy in expanded form The functions in this sub-menu expand vector/matrix results to several columns in the copied output
    • Copy Selection in expanded form copies the selected cells as a table in expanded form (see box at the end of this section!)
    • Copy Selection in expanded form, flipped copies the selected cells as a table in expanded form and rotates the table
    • Copy Selection in expanded form, w/o header copies the selected cells as a table in expanded form, but without row/column headers
    • Copy Selection in expanded form, w/o header, flipped copies the selected cells as a table in expanded form, but without row/column headers and rotates the table
    • Copy Selection in expanded form, to Matlab copies the selected cells as a table in expanded form into a Matlab script
    • Copy Selection in expanded form, to Matlab, flipped copies the selected cells as a table in expanded form into a Matlab script

Result statistics/Plots

You may view the contents of selected cells as histogram using these functions:

  1. histogram: rows-wise opens a histogram window that summarizes all selected values (vectors/matrizes are linearized) from each row into one histogram. So if you select the diffusion coefficients of species 1 and 2 from several runs, the window will contain two histograms, one for D1 and one for D2 overlayed. If you select data from several results (e.g. D1 and D2), QuickFit will ask whether to display all histograms in one window or to open one window per result.
  2. histogram: cells-wise works the same as above, but does not summarize the data from several runs. This is useful, if you get vectors/matrizes as results, where you want to compare the distributions in each of them. If you select single-number results, no histogram will be displayed, as it makes not much sense for one datapoint only.
  3. Result correlation will take the results in two selected rows and plot them against each other in a correlation plot.
