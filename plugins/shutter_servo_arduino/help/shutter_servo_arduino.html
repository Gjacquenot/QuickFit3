This plugin controls up to four servos connected to an Arduino board running a custom written C program with a simple protocoll, developed by Jan Krieger (B040, DKFZ).

The controllers are configured by a global configuration file shutter_servo_arduino.ini. It is stored in the global configuration directory. You may change some of the configuration options in the associated config directory which will show up, if - and only if - you have write access to this file. If the file does not exist, the plugin will load (but not write) to the file shutter_servo_arduino.ini either in the user config directory, or in the assets directory ([AssetsDir]/plugins/shutter_servo_arduino/) for the plugin.

In the shutter_servo_arduino.ini you will have to tell the plugin which stages are available and which serial ports they use. This is done inside a [shutterN] block for each shutter N (default values are underlined):

[shutterN]
port = port name (e.g. COM1 or /dev/tty0)
id = id of the shutter in the driver connected to the given port (0 | 1 | 2 | 3)
port_speed = port speed (9600 | 19200 | 38400 | ...)
port_databits = data bits used for the communication (5 | 6 | 7 | 8 | 9)
port_parity = parity bit (none | odd | even)
port_handshaking = handshaking mode (none | xonxoff | hardware)
port_stopbits = port speed (1 | 1.5 | 2)
port_timeout = port timeout in milliseconds (default: 500)
shutter_operation_duration = time for switching the shutter in milliseconds (default: 500)

The number of shutters is stored in the variable shutter_count.

Here is an example configuration file for windows and three servos on two controllers connected to COM1 and COM5:

shutter_count=3

[shutter1]
port=COM1
id=1
port_speed=9600
shutter_operation_duration=250

[shutter2]
port=COM1
id=2
shutter_operation_duration=500

[shutter3]
port=COM5
port_speed=19200 id=1
shutter_operation_duration=750
