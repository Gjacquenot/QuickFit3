You can enter a simple filter expression there. then only those files will be displayed, which contain the given expression. As an example, if your project contains a full FCCS measurement with three RDRs per measurement (ACF0,ACF1 and FCCS), but you only want to fit ACF0-RDRs with this evaluation, simply enter ACF0 into the left field. the rhs filter-field allows to exclude all records, that do NOT contain the specified filter string. In addition you can use regular expressions for the filter strings. $$faq_end$$ $$faq_start$$ What can I do if a multi-threaded imagingFCS fit freezes my QuickFit? $$faq_answer$$ This situation should not occur, but if: This may sometimes happen, if the multi-threaded fit ends up in an (unintended) dead-lock. You can do several things:
  1. set the maximum number of threads (in QuickFit's settings dialog) to a lower value (usually 2-4 works fine).
  2. if this happens for the newMT default variant of the fits, use the MT variant instead (click on the down-arrow on the rhs of the fit buttons to select a different method), which is slower but uses a different scheme for coordination between the different threads.
  3. use the slower non-threaded fit (click on the down-arrow on the rhs of the fit buttons to select a different method)
