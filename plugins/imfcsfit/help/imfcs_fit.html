This plugin allows to perform fits for imaging FCS data. It shares most functionality with the FCS Curve Fitting Plugin. Both plugins determine the best-fit parameters of a given theoretical FCS model function $$math:f(\tau;\vec{p})$$ from a measured correlation curve $$math:(\tau_i\hat{g}_i)_{i\in\mathbb{N}}$$. In order to perform this task, they solve a least-squares optimization problem: $$bmath:\vec{p}^\ast=\argmin_{\vec{p}}\ \sum_{i=1}^N\left(\frac{f(\tau_i;\vec{p})-\hat{g}_i}{\sigma_i}\right)^2\ \ \equiv\ \ \argmin_{\vec{p}}\ \sum_{i=1}^N\ \left[\hat{E}_i^\mathrm{(W)}\right]^2.$$ Here $$math:\sigma_i$$ are weighting factors (see Fit Weighting)

The major difference to thze standard FCS fit plugin is only is: The FCS fit plugin stores the fit results for each run in a separate column/row/set of fit results, whereas this plugin creates one set for the average run and one set for all other runs. The fit results are then stored as vectors/matrices inside this set. This saves a considerable amount of memory in the project file and while QuickFit is running. Also it improves the speed of execution of the imFCS raw data record Plugin.
This plugin allows to perform fits on several processor cores in parallel (multi-threaded). Set the maximum number of processors to use in the general settings dialog.
