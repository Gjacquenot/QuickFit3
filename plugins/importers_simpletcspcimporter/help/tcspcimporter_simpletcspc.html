$$qf_commondoc_header.start$$ $$qf_commondoc_header.end$$ This plugins imports photons from simple TCSPC files, defined as follows:
  1. The binary files contain a header which is not parsed at the moment. The header ends with a sequence of 32 bytes with a values of 0xFF each.
  2. Each photon records contains the duration since the last photon and is 64 bits long:
    1. bits 0..56 contain the time since the last photon in ticks (tick duration given as parameter and the user is asked for it on loading the file)
    2. bit 57 indicates photon in channel 2
    3. bit 58 indicates photon in channel 3
    4. bit 59 indicates photon in channel 1
    5. bit 60 indicates a special channel record for "RED OFF" (no photon)
    6. bit 61 indicates a special channel record for "RED ON" (no photon)
    7. bit 62 indicates a special channel record for "GREEN OFF" (no photon)
    8. bit 63 indicates a special channel record for "GREEN ON" (no photon)
$$note:THe 64-bit records are stored in the file in LITTLE ENDIAN form ... on a BIG ENDIAN system this will be corrected!$$
