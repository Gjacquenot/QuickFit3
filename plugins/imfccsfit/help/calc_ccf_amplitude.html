Function

This tool allows to calculate the relative CCF amplitude for imagingFCS raw data records in the project, i.e. it calculates an image where each pixel $$math:(x,y)$$ contain this information: $$bmath:\mbox{RCCF}(x,y)=\frac{\sum_{i=0}^{N-1} g_{\mbox{CCF}, (x,y)}(\tau_i)}{\sum_{i=0}^{N-1} g_{\mbox{ACF}, (x,y)}(\tau_i)}=:\frac{C}{A}$$ where $$math:g_{\mbox{CCF}, (x,y)}(\tau_i)$$ is the crosscorrelation amplitude at time lag $$math:\tau_i$$ and $$math:g_{\mbox{ACF}, (x,y)}(\tau_i)$$ is the autocorrelation amplitude at time lag $$math:\tau_i$$. The number $$math:N$$ gives the number of correlation channels over which to average in each pixel. The plugin also estimates an error for this quantity: $$bmath:\Delta\mbox{RCCF}(x,y)=\sqrt{\left(\Delta A\cdot\frac{C}{A^2}\right)^2+\left(\frac{\Delta C}{A}\right)^2}$$ Here the errors are defined as the average of the correlation functions' standard deviation: $$bmath:\Delta A=\frac{1}{N}\cdot\sum_{i=0}^{N-1}\sigma_{\mbox{CCF}, (x,y)}(\tau_i) $$ In all of these equations $$math:\tau_0$$ is the minimal lag time of the measured correlation functions.

User Interface

The tool can be opened by clicking in the main window on the menu entry Tools|imFCCS|calculate relative CCF amplitude. Subsequently a window will open up where a file for the autocorrelation and a file for the crosscorrelation can be selected, Buttons next to these file-selection comboboxes alles to move on to the next file to process. When a new ACF is selected the dialog tries to determine the fitting CCF file automatically. If both files are selected, overview images of the files are displayed below. A third plot shows the $$math:\mbox{RCCF}(x,y)$$ image calculated from the selected files.

The function of the single buttons:

  • "store result" will save the RCCF image in the CCF file.
  • "store all results" will walk through all files (basically by clicking "next ACF" and stores the resulting RCCF map. If for every measurement one CCF and two ACFs are present in the project, this will calculate two RCCFs (one for each ACF).
  • "store all results, same role" will do the same as above, but it will rather use "next file, same role". This function is useful,, when for every measurement one CCF and two ACFs are present in the project. Then
  • "next ACF": will select the next ACF file in the list (and the fitting CCF file
  • "next file": will select the next file in the list, without caring whether it contains an ACF or not (and the fitting CCF file
  • "next file, same role": will select the next file in the list with the same role, as the current file
