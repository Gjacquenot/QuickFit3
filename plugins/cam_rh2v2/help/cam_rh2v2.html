This plugin controls all cameras with HW-YAID based output streams. Currently RADHARD2 with custom firmware and correlation output and MULTISENSOR are supported.

Internally it uses a so called processing_chain: every single task is performed by a thread (USB data acquisition, frame analysis, preview, etc.), connected by FIFOs.

processing_chain depends on libusb-1.0

Configuration Options (RADHARD2)


Configuration dialog.
configuration option meaning
Frame accumulation count The number of frames that are accumulated before being displayed (applies for preview only).
Number of Frames to be read out Acquisition stops when this number of frames is reached. When temporal binning is enabled, the actual number of frames has to be multiplied by the binning value.
First line of ROI It is possible to define a region of interest. This number represents the first line of it. A ROI must contain at least two lines.
Last line of ROI see above.
Frame integration time This specifies the time between two subsequent frames.
Temporal binning The RADHARD2 firmware allows the accumulation of up to 3 subsequent frames before the data is sent to the correlator. As the sensors are limited to detect single photons only, this can improve the detection of multi-photon events.
Acquire HW-correlator data Wheather the hardware correlator is used in parallel. If not, the frame integration time can be choosen more freely.

USB Speed Considerations

In principal USB 2.0 speed is a crucial point in the whole setup. If frames are not transmitted completely, the correlation might become unreliable. For a single second measurement, a frame integration time of 5.7µs for full frames should be possible.
