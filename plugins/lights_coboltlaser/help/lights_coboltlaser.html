This plugin controls laser from ""Cobolt SE which are connected to a RS232 port on the computer.

The controllers are configured by a global configuration file lights_coboltlaser.ini. It is stored in the global configuration directory. You may change some of the configuration options in the associated config directory which will show up, if - and only if - you have write access to this file. If the file does not exist, the plugin will load (but not write) to the file lights_coboltlaser.ini either in the user config directory, or in the assets directory ([AssetsDir]/plugins/lights_coboltlaser/) for the plugin.

In the lights_coboltlaser.ini you will have to tell the plugin which remote boxes are available and which serial ports they use. This is done inside a [lightsourceN] block for each light source N (default values are underlined):

[lightsourceN]
port = port name (e.g. COM1 or /dev/tty0)
label = name of the laser
wavelength = wavelength of the laser in nm
max_power = maximum power of the laser in mW
type = type of the laser and it's generation'(DPSS4 | DPSS5 | DPSS6 | MLD6 )
power_send_factor = typically the laser power has to be sent in W, if this is not the case, this factor can be changed to calculate the power to send to the controller from the power (in mW) given to the plugin (0.001)
port_speed = port speed (115200 | 19200 | 38400 | ...)
port_databits = data bits used for the communication (5 | 6 | 7 | 8 | 9)
port_parity = parity bit (none | odd | even)
port_handshaking = handshaking mode (none | xonxoff | hardware)
port_stopbits = port speed (1 | 1.5 | 2)
port_timeout = port timeout in milliseconds (default: 500)

The number of lasers is stored in the variable lightsource_count.

Here is an example configuration file for Linux and one laser on /dev/ttyS0, if the laser is used on windows, simply replace /dev/ttyS0 by the name of the COM port it is connected to (e.g. COM1)):

lightsource_count=1

[lightsource1]
port=/dev/ttyS0
port_speed=115200
type=DPSS4
label="Calypso 491nm"
wavelength=491
max_power=35

See also the help page on Linux hardware enumeration with udev for further information
