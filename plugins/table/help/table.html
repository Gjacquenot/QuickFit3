    • Description

    This plugin allows the user to view/edit a table of values. It implements these features:

    • The values may be of diferent datatypes (integer or floating point number, text, ...).
    • Data may be loaded from or saved to external files.
    • Data exchange with other applications (Excel, Origin, ...) and other QuickFit plugins is usually possible via the clipboard.
    • Many common table editing functions are implemented (insert/delete rows/cols, table resize, ...).
    • A expression parser allows to calculate the contents of cells (or complete columns) in dependence of the contents of other cells.
    • You can access all editing functionality from the menu, a context-menu you reach by a right-click on the table and the buttons above the table
    • The columns may be formatted via the menu or by double-clicking on a column header. A dialog that allows to change its properties will pop up.
    $$see:Details on the table editor part of this plugin are described in this help page.$$

    In addition it is possible to create column-oriented plots from the data in that table. The curves in such a plot can also be used for data fittng:

    • regression analysis of plots
    • nonlinear fit to plots
    $$see:Details on the plot editor part of this plugin are described in this help page.$$

    Usage Notes & Hints

    • In order to add a table to the project, there are different ways:
      1. In QuickFit's main window, click on Data Items | Insert Raw Data | Table | Editable Table or ... | Tabel | Table from file to add a new empty table, or a read-only table, read from a file to the project.
      2. Most plots in QuickFit can be converted to a table. For that, right-click on the plot and choose copy plot to table RDR. This function will generate a new (or extend an existing) table record and add all data, that is associated with the plot. In addition the plot is also copied into the table RDR, so you can edit, or extend it. See $$qf_ui_jkqtplotter_helpfiletitle$$ for more information on this.
    • You may insert either an "Editable Table" or a "Table From File" into a QuickFit project. The lather one may NOT be edited by the user, as it simply represents the contents of a file. If you want to edit the contents of a file insert an "Editable Table" and then load the file using . The data of an Editable Table is stored inside the QuickFit project file, so such a raw data record does NOT depend on any external file. The data of a read-only table is typically stored in an external file, which has to be present along with the QuickFit project file, so the data can be displayed.
