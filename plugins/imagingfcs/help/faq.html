$$qf_commondoc_header.start$$ $$qf_commondoc_header.end$$ $$faq_start$$ What is an imaging fluorescence correlatioon spectroscopy (imFCS)? $$faq_answer$$ In standard FCS, measurements are performed for single positions in a sample. In imFCS, an array detector (fast camera) is used to record the fluorescence fluctuations from several pixels in the sample simultaneously. The for each pixel in the measurement, an autocorrelation (and possibly pixel-pixel or two-color-cross-correlation) curves are calculated and evaluated. Finally the fit parameters (e.g. the diffusion coefficient) can be represented as a map or image for the sample, reporting on how fast particles move at different positions in the sample. imFCS is usually impelemnted on a total internal reflection fluorescence microscope (TIRF) or a single plane illumination microscope (SPIM). $$faq_end$$ $$faq_start$$ How can I correlate an image series in QuickFit for an imFCS data evaluation? $$faq_answer$$ The help page Correlating imFCS data and the tutorials imFCS analysis of beads in solution and imFCS analysis in a cell give a step-by-step introduction to this task. $$faq_end$$ $$faq_start$$ Which image stack data formats are supported by QuickFit? $$faq_answer$$ QuickFit supports all image stack data formats, for which a reader plugin is installed. QuickFit generates one raw data record (RDR) for each of these curves (all in the same RDR group) and an imFCCS global fit may recognize these several records and fit them all together. If you open the FCCS record (i.e. the RDR editor by double-clicking it in the project tree), you can choose the FCCS display mode:
This mode is suitable for 2-color FCCS data, since it will display the autocorrelation curves ACF0 (green), ACF1 (red) and the cross-correlation curve FCCS (blue) for each pixel in the correlation fucntion plot on the lhs. In addition you can provide a cross-talk and QuickFit will display crosstalk-corrected correlation amplitudes as lines in the plots and as number in the parameter panel on the rhs of the dialog (see screenshot above!).

For 2-pixel cross-correlation data (also often called distance cross-correlation data), you can choose the DCCF mode, which is more general than the FCCS mode. It displays for each pixel all correlation curves in the currenzt group, but it lacks the possibility to calculate cross-talk corrected CF amplitudes, since this does not make sense in all settings.
