Image Cropping

You can select a sub-region (region of interest, ROI) in your image series for which to do the correlation. All pixels outside the ROI will be ignored during processing. The ROI is specified by the first and last USED pixels (0-indexed) in both directions. So a range of x=0..3, y=2..5 will result in a 4×4 image.

Image Binning

This plugin supports two modes of binning:

  1. "normal" binning: The plugin will sum up the values from regions of nbinning×nbinning pixels. The regions are positioned non-overlapping on the input image (see image below). Note that in this mode some piixels at the right and bottom border of the image might be omitted!
  2. "interleaved" binning: The plugin will also sum up the values from regions of nbinning×nbinning pixels. The regions are positioned overlapping on the input image as shown in the figure below. This results in more correlation functions calculated from the same input image, but they are no longer statistically independent over several pixels, as they are calculated from input data that is in parts common to several neighboring pixels.
Note: It is possible to change the functionality from binning, i.e. summing up neighboring pixels: $$bmath:I_{ROI}(x,y)=\sum_{\xi=0..n_{binning}-1}\sum_{\mu=0..n_{binning}-1}I_{in}(x+\xi, y+\mu).$$ to averaging $$bmath:I_{ROI}(x,y)=\frac{1}{n_{binning}\cdot n_{binning}}\sum_{\xi=0..n_{binning}-1}\sum_{\mu=0..n_{binning}-1}I_{in}(x+\xi, y+\mu).$$ This might improve the correlation accuracy for high input values per pixel, as overflow/data representation errors in the correlators is reduced, see also notes on correlator types.
