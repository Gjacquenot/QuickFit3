The image stack plugin provides math parser functions, that allow to access the image stack properties and data of any image stack record in the project. Eachs such record, provides a set of image stacks (numbered 0, 1, ..., imagestack_getstacks(rdr_id)-1). Each stack has its own number of frames (imagestack_getframes(rdr_id, stack)), which all have the same size (imagestack_getwidth(rdr_id, stack), imagestack_getheight(rdr_id, stack)) and number of color-channels (imagestack_getchannels(rdr_id, stack)).

You can access the data of any frame in row-major form with the function imagestack_getframe(rdr_id, stack=0, frame=0, channel=0), or imagestack_getstack(rdr_id, stack=0, channel=0). Both functions retunr the data as a vector in row-major form, i.e. the image data I(x,y,f,ch) is returned as I(x=0,y=0,f=0), I(1,0,0),...,I(width-1,0,0),I(0,1,0),I(1,1,0),...,I(width-1,height-1,1).

$$note:These functions may also be applicable to other RDRs, which expose the image stack interface (e.g. imagingFCS records)!$$ $$faq_end$$
