Image Acquisition

This is the simples acquisition. It uses the acquisition functionality implemented in the camera drivers, so you should set the length of the image sequence and all parameters there (in the camera configuration, use ). If you select two cameras, the acquisitions will be started at nearly the same time. If you need exact timing, you will have to implement it using the trigger functionality of your cameras!

The data is written to files with a given prefix, which may also contain a path, e.g. c:/data/2011_08_25/cam1_%counter%. In the given example the files c:/data/2011_08_25/cam1_0.tif and c:/data/2011_08_25/cam1_0configuration.inic ould be created where the first one contains the acquired images and the second one all the available meta-data (exposure times, objectives, ...). In the prefix strings these special sequences are replaced:

  • %counter% is replaced by the value of the counter (which is increased after each acquisition!)

It is also possible to save a preview image as an overview (e.g. when you just acquire a region of interest of your camera). If this functionality is enabled an additional 16-bit (+ possibly 32-bit) TIFF file is recorded, using the preview camera configuration. The files are save as [prefix]_overview.tif and [prefix]_overview.32.tif. The 32-bit TIFF is created only if the camera returns 32-bit data. In that case the 16-bit TIFF file contains a down-scaled version of the image (highes gray-value is set to 65535 and all others are scaled accordingly!

After setting all options, you just have to click on and wait until the acquisition is complete. If there are any errors, they will be displayed as a message box and in the logging pane at the bottom of the window.
