Acquisition configuration files configuration.ini

Basic format & Example

The B040 SPIM plugin creates .configuration.ini files for every (data/image) acquisition, that is performed. These are files in the INI file format, i.e. generally consist of key/value pairs and groups: 

    [acquisition]
    acquisition\accumulation_time=0.00056
    acquisition\binning_horizontal=1
    acquisition\binning_vertical=1
    acquisition\bit_depth=14
    acquisition\camera_controller_card=CCI-23
    ...
    type=acquisition
    pixel_width=0.4
    pixel_height=0.4
    camera_pixel_width=24
    camera_pixel_height=24
    camera_model=DU860_BV
    sensor_model=DU860_BV
    magnification=60

    [setup]
    filterchangers\detection\filter=0
    filters\detection_filterchanger\manufacturer=Semrock
    filters\detection_filterchanger\name=Edge Basic 488 LP
    filters\detection_filterchanger\type=long pass
    filters\detection_splitter\manufacturer=???
    filters\detection_splitter\name=50:50 splitter
    filters\detection_splitter\type=neutral beam splitter
    laser1\line1\enabled=true
    laser1\line1\measured_power=10
    laser1\line1\name=491nm
    laser1\line1\set_power=10
    laser1\line1\unit=mW
    laser1\shutter=true

    cell=5

Common entries

The files are mostly split into these larger groups:

  • [acquisition] describes the setup of the microscope and cameras during the acquisition. If during an acquisition, several images/image series are taken, the settings for each of these are typically stored here. The settings during the main acquisition are then stored with the prefix acquisition\, the settings for taking a background image series with the prefix background\, for overview images under overview\ and so on.
  • [experiment] contains general describing information on the experiment, as they can be entered in the "experiment description" tab.
  • [files] contains a list of files, that belong to this acquisition. For each file, the relative path (name), the file type and a description are given.

These files are used e.g. by the imagingFCS Correlator to determine the pixel size, framerate etc.

Measurements files measured.dat

These are comma-separated values files (dot '.' as decimal separator), which contain measurements, such as temperature readings or laser powers, acquired during an image acquisition. The first column is the approximate time (in seconds, since start of the acquisition), at which the data was acquired.
