Introduction

This plugin controls a resistor heater (PI loop) controller running a custom written C program with a simple protocoll, developed by Jan Krieger (B040, DKFZ).

Configuration

The controllers are configured by a global configuration file meas_b040resheater.ini. It is stored in the global configuration directory. You may change some of the configuration options in the associated config directory which will show up, if - and only if - you have write access to this file. If the file does not exist, the plugin will load (but not write) to the file meas_b040resheater.ini either in the user config directory, or in the assets directory ([AssetsDir]/plugins/meas_b040resheater/) for the plugin.

In the meas_b040resheater.ini you will have to tell the plugin which stages are available and which serial ports they use. This is done inside a [deviceN] block for each device N (default values are underlined):

[deviceN]
port = port name (e.g. COM1 or /dev/tty0)
port_speed = port speed (115200 | 19200 | 38400 | ...)
port_databits = data bits used for the communication (5 | 6 | 7 | 8 | 9)
port_parity = parity bit (none | odd | even)
port_handshaking = handshaking mode (none | xonxoff | hardware)
port_stopbits = port speed (1 | 1.5 | 2)
port_timeout = port timeout in milliseconds (default: 500)

The number of devices is stored in the variable device_count.

Here is an example configuration file for windows and three servos on one controller connected to COM1:

device_count=1


[device1]
port=COM1
port_speed=9600

See also the help page on Linux hardware enumeration with udev for further information
